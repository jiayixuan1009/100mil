# BiyaPay 增长项目主执行计划

## 最终目标

在不刷量、不污染数据、不牺牲生产稳定性的前提下，将 BiyaPay 网站体系的月真实可运营流量提升到 100 万级，并建立可持续复用的增长执行系统。

这里的“真实可运营流量”定义为：

- 可以在 GA4 / GSC 中被稳定识别和解释。
- 不属于测试环境、预发环境、内网、本地、异常爬虫或明显机器流量。
- 能进入注册、KYC、入金、交易等后续业务判断链路。

## Agent 工作方式

本项目按 Agent 模式推进：

- 优先直接执行，不停留在建议层。
- 非权限、风险、重大歧义问题不频繁向用户确认。
- 每个阶段结束后简要汇报已完成内容、当前结果和下一步。
- 如原计划不合理，可基于数据自行调整，并说明原因。
- 每一阶段和专项都必须形成文档和交付物。

## 项目总路径

### 路径 1：数据可信化

先建立可信数据底座，否则后续所有增长动作都可能建立在错误口径上。

阶段包含：

- GA4 hostname 清洗。
- Direct 流量专项。
- 跨域归因复核。
- 漏斗事件审计。

核心交付：

- 真实流量口径。
- 污染流量排除规则。
- 漏斗缺口清单。

### 路径 2：现有页面提效

从现有 GSC 曝光和排名资产中拿最快增量。

阶段包含：

- GSC Top 200 机会页提取。
- 高曝光低 CTR 页面改造。
- Top 30 高优先 URL 先执行。
- 模板页首轮提效。

核心交付：

- 机会 URL 清单。
- 页面优先级和动作表。
- 第一批页面改造结果。

### 路径 3：技术与模板治理

控制大规模页面站点的结构性损耗。

阶段包含：

- indexability / canonical / hreflang / orphan / redirect / CWV 治理。
- compare / converter / sendmoney / bank code / iban / swift 模板升级。

核心交付：

- 技术问题优先级表。
- 模板规格文档。
- 模板改造排期。

### 路径 4：内容、权威与 AI 可见性增长

在技术和现有页面提效之后，再扩大商业意图覆盖和 AI 搜索引用能力。

阶段包含：

- 商业意图内容集群。
- 竞品对比、汇款、USDT 出金、银行工具、汇率换汇主题扩展。
- 外链、品牌引用、AI 搜索实体与证据源建设。

核心交付：

- 内容集群地图。
- 外链与引用目标清单。
- AI 可见性跟踪。

## 当前执行顺序

### 第一阶段

先做：

- Direct 流量专项。
- GA4 hostname 清洗。
- 转化事件审计。
- GSC Top 200 机会页提取。
- Top 200 技术状态补齐。

原因：

- 这是最短路径，能在 7-10 天内同时拿到“可信口径”和“可执行页面清单”。

### 第二阶段

接着做：

- Top 30 页面优化。
- compare / converter / sendmoney 模板诊断。
- 关键技术问题修复。

### 第三阶段

随后做：

- 模板规格升级。
- 商业意图内容扩展。
- AI 搜索/GEO 专项。

## 交付机制

每个阶段必须产出以下内容：

- 一个施工文档。
- 一个结果报告或模板。
- 一个结构化表格清单或 CSV 规范。
- 下一阶段输入条件。

## 项目完成标准

项目不是以“做了多少 SEO 动作”为完成标准，而是以以下结果为标准：

- 关键数据口径稳定。
- Direct 流量异常可解释。
- Top 200 SEO 机会页已分层并完成首批执行。
- 模板页质量显著提升。
- SEO 页面可以追踪到注册、KYC、入金、交易链路。
- 月真实可运营流量持续逼近并达到 100 万级。
